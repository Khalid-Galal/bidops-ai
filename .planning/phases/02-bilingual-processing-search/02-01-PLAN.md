3. **Create `app/services/text_processing/arabic_normalizer.py`**:
   - Implement `normalize_arabic(text: str) -> str` function that applies these normalizations IN ORDER:
     a. Remove tashkeel (diacritics): strip Unicode range \u064B-\u0652 using regex
     b. Normalize alef variants: replace \u0622, \u0623, \u0625 with bare alef \u0627
     c. Normalize teh marbuta: replace \u0629 with heh \u0647
     d. Convert Eastern Arabic numerals (U+0660-U+0669) to Western digits (0-9)
     e. Normalize whitespace (collapse multiple spaces, strip leading/trailing)
   - Use PyArabic's `strip_tashkeel()` for diacritics removal (more reliable than regex for edge cases). Fall back to regex if PyArabic is unavailable.
   - Include `normalize_for_search(text: str) -> str` that applies `normalize_arabic()` followed by `.lower()` for search indexing consistency. This function MUST be used at both index time and query time.
   - Add docstrings explaining each normalization step and WHY it matters for search (e.g., "Users may search with or without diacritics; normalizing ensures matches").

4. **Update `requirements.txt`**:
   - Add these new dependencies (with comments):
     ```

     PyArabic
     python-bidi
     ```
   - Do NOT add all Phase 2 deps yet -- only what this task needs. Other deps will be added in Plans 02-02 and 02-03.
  </action>
