<objective>
Build the hybrid search layer combining BM25 keyword search with ChromaDB semantic search, exposed via a REST API endpoint.

Purpose: Enable users to search across all ingested project documents by keyword (exact term matching) or by meaning (semantic similarity), with results ranked by Reciprocal Rank Fusion for optimal relevance. This is the user-facing search capability that consumes the indexing infrastructure built in Plan 02-02.

Output: KeywordSearchService (BM25), VectorSearchService (ChromaDB), HybridSearchService (RRF fusion), search API endpoint, and Pydantic schemas.

2. **Create `app/services/search/vector_search.py`**:
   - Implement `VectorSearchService` class:
     - Constructor takes `embedding_service: EmbeddingService` (from Plan 02-02)
     - `search(project_id: int, query: str, n_results: int = 20, language_filter: str | None = None) -> list[tuple[str, str, dict, float]]`:
       - Apply `normalize_for_search()` to query (CRITICAL: same normalization as indexing, Pitfall 6)
       - Call `embedding_service.get_collection(project_id)`
       - Build `where` filter for language if specified
       - Call `collection.query(query_texts=[normalized_query], n_results=n_results, where=where, include=["documents", "metadatas", "distances"])`
       - Return list of `(chunk_id, document_text, metadata_dict, distance_score)` tuples
       - Handle empty collection gracefully (return empty list)

3. **Create `app/services/search/keyword_search.py`**:
   - Implement `KeywordSearchService` class:
     - Constructor: no params
     - Internal cache: `_indices: dict[int, tuple[BM25Okapi, list[str], list[str], list[dict]]]` mapping project_id to (bm25, chunk_ids, documents, metadatas)
     - `build_index(project_id: int, embedding_service: EmbeddingService) -> None`:
       - Fetch ALL chunks from the project's ChromaDB collection using `collection.get(include=["documents", "metadatas"])`
       - Tokenize: apply `normalize_for_search()` THEN `.split()` to each document text (CRITICAL: must match query normalization)
       - Build `BM25Okapi(tokenized_docs)`
       - Cache in `_indices`
     - `invalidate_index(project_id: int) -> None`: Remove project from cache. Call this whenever new documents are indexed (Pitfall 4).
     - `search(project_id: int, query: str, embedding_service: EmbeddingService, top_k: int = 20) -> list[tuple[str, float, str, dict]]`:
       - If project not in cache, call `build_index()`
       - Tokenize query: `normalize_for_search(query).split()`
       - Get BM25 scores: `bm25.get_scores(tokenized_query)`
       - Sort by score descending, take top_k where score > 0
       - Return list of `(chunk_id, bm25_score, document_text, metadata_dict)` tuples
     - Import `from rank_bm25 import BM25Okapi`
     - Import `from app.services.text_processing import normalize_for_search`

4. **Create `app/services/search/hybrid_search.py`**:
   - Define `SearchResult` dataclass:
     - `chunk_id: str`
     - `text: str` -- the chunk text for display
     - `score: float` -- combined RRF score
     - `document_id: int`
     - `page_number: int`
     - `language: str`
     - `filename: str`
     - `chunk_type: str` -- "text" or "table"
     - `section_name: str | None`
   - Implement `HybridSearchService` class:
     - Constructor takes:
       - `embedding_service: EmbeddingService`
       - `alpha: float = 0.7` (weight for semantic, so 0.3 for keyword)
       - `rrf_k: int = 60` (standard RRF constant)
     - Internal: `keyword_service = KeywordSearchService()`
     - Internal: `vector_service = VectorSearchService(embedding_service)`
     - `search(project_id: int, query: str, top_k: int = 10, mode: str = "hybrid") -> list[SearchResult]`:
       - If mode == "semantic": only vector search, return results
       - If mode == "keyword": only keyword search, return results
       - If mode == "hybrid":
         1. Run vector search with `n_results = top_k * 3` (over-retrieve for fusion)
         2. Run keyword search with `top_k = top_k * 3`
         3. Apply RRF fusion: for each chunk, `score = alpha / (rrf_k + semantic_rank + 1) + (1-alpha) / (rrf_k + keyword_rank + 1)`. Chunks appearing in only one list get score from that list only.
         4. Sort by combined score, take top_k
         5. Build SearchResult objects from metadata
       - Return list[SearchResult] sorted by score descending
     - `invalidate_keyword_index(project_id: int) -> None`: delegate to keyword_service.invalidate_index(). Call after new documents indexed.
     - `_rrf_fusion(semantic_results, keyword_results, top_k) -> list[SearchResult]`: private method implementing the RRF algorithm described above

5. **Update `requirements.txt`**:
   - Add:

     ```
     rank-bm25
     ```
