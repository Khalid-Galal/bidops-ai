1. **Create `app/schemas/extraction.py`**:
   - Define Pydantic models for structured extraction output:
     - `Citation(BaseModel)`:
       - `document_name: str` -- Field(description="Filename of the source document")
       - `page_number: int` -- Field(ge=1, description="1-based page number where the value was found")
       - `quote: str` -- Field(min_length=1, description="Exact verbatim quote from the source supporting this value")
     - `ExtractedField(BaseModel)`:
       - `value: str | None` -- Field(default=None, description="The extracted value, or null if not found in documents")
       - `confidence: float` -- Field(ge=0.0, le=1.0, description="Confidence score from 0.0 to 1.0")
       - `confidence_level: str` -- Field(default="low", description="Human-readable confidence: high, medium, or low")
       - `citations: list[Citation]` -- Field(default_factory=list, description="Source citations supporting this extraction")
       - `reasoning: str` -- Field(default="", description="Brief explanation of how the value was determined")
       - `requires_review: bool` -- Field(default=True, description="Whether this extraction needs human review")
     - `ProjectSummary(BaseModel)` -- all 13 fields as ExtractedField:
       - `project_name: ExtractedField`
       - `project_owner: ExtractedField`
       - `location: ExtractedField`
       - `submission_deadline: ExtractedField`
       - `bid_validity_period: ExtractedField`
       - `pre_bid_meeting_date: ExtractedField`
       - `scope_of_work: ExtractedField`
       - `contract_type: ExtractedField`
       - `tender_bond: ExtractedField`
       - `advance_payment: ExtractedField`
       - `retention_percentage: ExtractedField`
       - `payment_terms: ExtractedField`
       - `stakeholders: ExtractedField`
     - `ExtractionResponse(BaseModel)` -- API response wrapper:
       - `project_id: int`
       - `status: str` -- "completed", "in_progress", "failed"
       - `summary: ProjectSummary | None`
       - `extraction_time_seconds: float | None`

     - `citations: list[Citation]` -- Field(default_factory=list, description="Source citations")

2. **Create `app/services/llm/gemini_service.py`**:
   - Import `instructor`, `from pydantic import BaseModel`, `logging`, `from tenacity import retry, stop_after_attempt, wait_exponential, retry_if_exception_type`
   - Implement `GeminiService` class:
     - `__init__(self, api_key: str, model: str = "gemini-2.5-pro")`:
       - Store `_api_key`, `_model`
       - `_client = None` (lazy init)
     - `_get_client(self)`:
       - If `_client is None`:
         - `self._client = instructor.from_provider(f"google/{self._model}", api_key=self._api_key)`
       - Return `self._client`
     - `extract(self, prompt: str, response_model: type[BaseModel], max_retries: int = 3) -> BaseModel`:
       - Docstring: "Extract structured data from prompt using Gemini with instructor validation."
       - Get client via `_get_client()`
       - Call `client.create(messages=[{"role": "user", "content": prompt}], response_model=response_model, max_retries=max_retries)`
       - Return the response (already validated Pydantic model)
       - Wrap in try/except for common errors:
         - `Exception` -> log error, re-raise with context
       - Add `@retry` decorator from tenacity for transient API errors (rate limits):
         - `retry=retry_if_exception_type((ConnectionError, TimeoutError))`
         - `stop=stop_after_attempt(3)`
         - `wait=wait_exponential(multiplier=1, min=2, max=30)`
         - NOTE: instructor already handles validation retries internally. The tenacity retry here is ONLY for network-level transient errors (ConnectionError, TimeoutError). Do NOT retry on ValueError or ValidationError -- those are legitimate schema failures that instructor handles.

3. **Create `app/services/llm/context_builder.py`**:
   - Import `from __future__ import annotations`, `from app.services.search.hybrid_search import SearchResult` (TYPE_CHECKING guard), `from app.services.extraction.field_definitions import FieldDefinition`
   - Implement `build_labeled_context(chunks: list[SearchResult]) -> str`:
     - Docstring: "Build context string with labeled source chunks for LLM attribution."
     - For each chunk, create label: `[SOURCE:{chunk.filename} | PAGE:{chunk.page_number}]`
     - Concatenate: `{label}\n{chunk.text}`
     - Join with `\n\n---\n\n` separator
     - Return the full context string
   - Implement `build_extraction_prompt(field_def: FieldDefinition, context: str) -> str`:
     - Docstring: "Build extraction prompt for a specific field with source context."
     - Return a multi-line f-string prompt:
       ```
       Extract the following field from the tender document excerpts below.

       FIELD: {field_def.name}
       DESCRIPTION: {field_def.description}
       EXPECTED TYPE: {field_def.field_type}
       {f"VALID VALUES: {', '.join(field_def.enum_values)}" if field_def.enum_values else ""}

       INSTRUCTIONS:
       1. Find the {field_def.name} in the provided document excerpts.
       2. Copy the EXACT value as it appears in the source document.
       3. For the quote field in citations, copy the EXACT sentence(s) containing this value -- do NOT paraphrase or summarize.
       4. Include the document_name and page_number from the [SOURCE:... | PAGE:...] labels in your citation.
       5. If the field is not found in any excerpt, set value to null and confidence to 0.0.
       6. NEVER fabricate or infer values not explicitly stated in the documents.
       7. For list-type fields (e.g., stakeholders), include ALL items found across all excerpts.
       8. Set confidence between 0.0 and 1.0 based on how clearly and explicitly the value appears.

       DOCUMENT EXCERPTS:
       {context}
       ```
     - If `field_def.field_type == "list"`, append extra instruction: "Return the value as a comma-separated list of items."
     - If `field_def.field_type == "date"`, append: "Preserve the date format exactly as it appears in the source (do not convert or reformat)."
