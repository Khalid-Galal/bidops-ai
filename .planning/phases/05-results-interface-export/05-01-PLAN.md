   - `index: int` (0-based index within the category list)
   - `updates: dict` (fields to update, e.g. `{"checked": true}` or `{"requirement": "edited text"}`)

2. Add the PATCH endpoint `@router.patch("/projects/{project_id}/checklist/items")`:
   - Open async session, get Project by ID
   - 404 if project not found or checklist_json is None
   - Parse checklist_json with `json.loads()`
   - Validate category exists in parsed data
   - Validate index is within bounds of the category list
   - Apply `updates` dict to the specific item: `category_items[index].update(update.updates)`
   - Re-serialize with `json.dumps(checklist_data, ensure_ascii=False)`
   - Assign back to `project.checklist_json`
   - Use `flag_modified(project, "checklist_json")` from `sqlalchemy.orm.attributes` to ensure SQLAlchemy detects the mutation
   - `await session.commit()`
   - Return `{"status": "updated", "category": update.category, "index": update.index}`

3. Add required imports at top: `import json`, `from pydantic import BaseModel`, `from sqlalchemy.orm.attributes import flag_modified`

Do NOT modify any existing endpoints. Only add the new model and PATCH endpoint.
