  <action>
Add a Search tab to the existing tab navigation in project.html (created by Plan 01) and append search-specific CSS styles.

**project.html changes:**

1. Add a new tab button after the Checklist tab button:
   ```html
   <button class="tab" data-tab="search">Search</button>
   ```

2. Add Search tab content panel after the checklist tab-content div:
   ```html
   <div class="tab-content" id="tab-search">
   ```

3. Search form inside the panel -- a card containing:
   - Search input: `<input type="text" id="search-input" placeholder="Search documents..." class="search-input">` with autofocus when tab is first activated
   - Mode selector: `<select id="search-mode" class="search-mode-select">` with three options:
     - `<option value="hybrid" selected>Hybrid (Keyword + Semantic)</option>`
     - `<option value="semantic">Semantic (Meaning)</option>`
     - `<option value="keyword">Keyword (Exact Match)</option>`
   - Search button: `<button class="btn btn-primary" id="search-btn">Search</button>`
   - Result count: `<span id="search-count" class="search-count"></span>`
   - Arrange input and button in a flex row with the select below or beside depending on screen width.

4. Results container: `<div id="search-results" class="search-results"></div>` initially empty.

5. JavaScript additions (inside the existing script block):

   - `performSearch(projectId)`:
     - Get query from `#search-input`, mode from `#search-mode`
     - If query is empty, show message "Enter a search query" and return
     - Disable search button, show "Searching..."
     - fetch GET `/api/projects/${projectId}/search?q=${encodeURIComponent(query)}&mode=${mode}&limit=20`
     - On success, render results into `#search-results`
     - Update `#search-count` with "N results found"
     - Re-enable button
     - On error, show error message

   - `renderSearchResults(results)`:
     - If results array is empty, show "No results found. Try a different query or search mode."
     - For each result, create a search result card with:
       - Text content (the `text` field) -- show full text, no truncation
       - Source metadata bar: filename, page number, language badge, chunk type badge, relevance score
       - Language badge: "AR" for Arabic (with `dir="rtl"` on the text), "EN" for English, "MIXED" for mixed
       - Chunk type badge: "TABLE" or "TEXT"
       - Score displayed as percentage: `Math.round(result.score * 100) + '%'`
     - If text contains Arabic content (language === "ar"), apply `dir="rtl"` and `class="rtl-text"` to the text container

   - Event listeners:
     - `#search-btn` click -> `performSearch(projectId)`
     - `#search-input` keypress (Enter key) -> `performSearch(projectId)`
     - No debounce needed since search is triggered by explicit action (button or Enter), not on-type

   - The search tab does NOT need lazy loading like summary/checklist -- it starts empty and the user initiates searches.
