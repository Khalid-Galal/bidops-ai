**Part A: Create `app/api/checklist.py`**

Follow the EXACT same structure as `app/api/extraction.py`. This is a near-mirror with s/extraction/checklist/ substitutions. Use `from __future__ import annotations`, logging, FastAPI imports.

1. **Lazy singleton `_checklist_service`** -- Create `_get_checklist_service()` following the exact pattern from `_get_extraction_service()` in extraction.py:
   - Check `_checklist_service is None`
   - Get settings, validate gemini_api_key
   - Import and create: EmbeddingService, HybridSearchService, GeminiService, CitationVerifier
   - Create `ChecklistService(search_service=search_svc, llm_service=llm_svc, citation_verifier=verifier)`
   - Return singleton

   NOTE: This creates separate service instances from the extraction API. That is fine -- they are lightweight wrappers. The expensive models (NLI, embedding) are loaded lazily by their respective services and would be shared at the model level if the same model name is used. But for simplicity and isolation, separate instances are acceptable.

2. **Helper `_count_checklist(checklist: RequirementsChecklist) -> tuple[int, int]`**:
   - Count `total_requirements` = len(requirements) + len(submission_documents) + len(eligibility_criteria)
   - Count `requirements_requiring_review` = count where requires_review == True across all three lists
   - Return (total, requiring_review)

3. **`router = APIRouter(tags=["checklist"])`**

4. **POST `/projects/{project_id}/checklist`** -- response_model=ChecklistResponse:
   - Verify project exists (404 if not)
   - Check `project.checklist_status == "in_progress"` -> 409 Conflict
   - Get checklist_service singleton
   - Call `await checklist_service.extract_and_persist_checklist(project_id)`
   - Count totals
   - Return ChecklistResponse with status="completed"
   - On exception: log, raise 500

5. **GET `/projects/{project_id}/checklist`** -- response_model=ChecklistResponse:
   - Verify project exists (404 if not)
   - If `checklist_status is None` -> return status="not_started"
   - If `checklist_status == "in_progress"` -> return status="in_progress"
   - If `checklist_status == "completed"` and `checklist_json` exists -> parse with `RequirementsChecklist.model_validate_json()`, count totals, return
   - Else -> return status="failed"

   Follow the exact same status handling pattern as `get_extraction_result()` in extraction.py.
