  <name>Task 1: Create ChecklistService with multi-query retrieval and category extraction</name>
  <files>app/services/extraction/checklist_service.py</files>
  <action>
Create `app/services/extraction/checklist_service.py` following the same structural pattern as `extraction_service.py`. Use `from __future__ import annotations`, TYPE_CHECKING guards for service dependencies, logger setup.

**Constructor** -- takes the same 3 services as ExtractionService:
- `search_service: HybridSearchService`
- `llm_service: GeminiService`
- `citation_verifier: CitationVerifier`

Additionally, store an optional `embedding_service` reference for deduplication (loaded lazily from search_service). The constructor should NOT take EmbeddingService as a parameter. Instead, access it via `self._search_service._embedding_service` when needed for deduplication (this is how the codebase accesses it -- no need for a new dependency injection).

**Method: `_retrieve_category_chunks(project_id, category) -> list[SearchResult]`**

For each query in `category.queries`, call `self._search_service.search(project_id, query, top_k=category.top_k_per_query, mode="hybrid")`. Merge results across queries, deduplicate by `chunk.chunk_id` (keep first seen). Sort all unique chunks by score descending. Return top `category.max_context_chunks` chunks.

**Method: `_extract_category(project_id, category) -> list[VerifiedRequirement]`**

1. Call `_retrieve_category_chunks()` to get chunks.
2. If no chunks, return empty list (log info, do not raise).
3. Build labeled context via `build_labeled_context(chunks)`.
4. Build prompt via `build_checklist_extraction_prompt(category, context)`.
5. Extract via `await asyncio.to_thread(self._llm_service.extract, prompt=prompt, response_model=CategoryExtractionResponse)`.
6. Wrap in try/except -- on failure, log warning and return empty list (graceful degradation, same as ExtractionService).
7. For each `RequirementItem` in `response.items`, verify and convert to `VerifiedRequirement`:
   - Create a `Citation(document_name=item.source_document, page_number=item.page_number, quote=item.quote)`
   - Find matching source chunk using same logic as CitationVerifier._find_source_chunk (match by filename + page, fallback to filename only)
   - If source found: call `self._citation_verifier.verify_citation(claim=item.quote, source_text=source.text)` for NLI score
   - If no source found: nli_score = 0.0
   - Get retrieval score: score of the matched chunk, or 0.5 default
   - Call `self._citation_verifier.calculate_confidence(llm_confidence=item.confidence, retrieval_score=retrieval_score, nli_entailment_score=nli_score, has_verified_citation=nli_score >= 0.3)` to get (score, level, requires_review)
   - Build `VerifiedRequirement` with category=category.name and all computed fields
8. Return list of VerifiedRequirement objects.

**Method: `_deduplicate(requirements) -> list[VerifiedRequirement]`**

Semantic deduplication across categories using cosine similarity on the existing embedding model.

1. If len(requirements) <= 1, return as-is.
2. Get the sentence-transformers model from `self._search_service._embedding_service._get_model()` (the SentenceTransformer model used for embeddings).
3. Encode all requirement texts: `model.encode([r.requirement for r in requirements])`.
4. For each pair, compute cosine similarity. If similarity >= 0.9, mark the lower-confidence item as duplicate.
5. Return only non-duplicate items.
6. Log how many duplicates were removed.

IMPORTANT: Use numpy for cosine similarity (`np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))`). Import numpy as np.

If embedding model is not available (import error or service not loaded), skip deduplication and return all items with a warning log.

**Method: `_assemble_checklist(requirements) -> RequirementsChecklist`**

Split VerifiedRequirement list into three groups:
- `submission_documents`: category == "submission_documents"
- `eligibility_criteria`: category == "eligibility"
- `requirements`: everything else (technical, commercial, legal, hse)

Count totals:
- `total_count` = len(all items)
- `mandatory_count` = count where is_mandatory == True
- `categories_extracted` = sorted unique list of categories present

Return `RequirementsChecklist(...)`.

**Method: `async extract_checklist(project_id) -> RequirementsChecklist`**

Main orchestration method:
1. `start_time = time.time()`
2. Loop through `CHECKLIST_CATEGORIES`:
   a. Call `await self._extract_category(project_id, category)`
   b. Append results to `all_requirements` list
   c. Log: "Extracted {N} {category.display_name} requirements"
   d. `await asyncio.sleep(0.5)` between categories (rate limiting)
3. Deduplicate: `unique = self._deduplicate(all_requirements)`
4. Assemble: `checklist = self._assemble_checklist(unique)`
5. Log: "Checklist extraction complete: {total_count} requirements in {elapsed:.1f}s"
6. Return checklist.

**Method: `async extract_and_persist_checklist(project_id) -> RequirementsChecklist`**

Same pattern as ExtractionService.extract_and_persist():
1. Set `project.checklist_status = "in_progress"` in DB.
2. Call `self.extract_checklist(project_id)`.
3. On success: store `project.checklist_json = checklist.model_dump_json()`, set `project.checklist_status = "completed"`.
4. On failure: set `project.checklist_status = "failed"`, re-raise.
5. Import from `app.database` and `app.models.project` inline (same pattern as extraction_service.py).

NOTE: The Project model will be modified in Plan 03 to add `checklist_json` and `checklist_status` columns. This service writes to those columns. The persist method should follow the exact same pattern as ExtractionService.extract_and_persist() with async session management.
