    1. Create `app/services/progress.py` -- simple in-memory progress store:
       - Module-level dict: `progress_store: dict[str, dict] = {}`
       - Functions:
         - `init_progress(task_id: str, total: int)` -- sets initial state: {"status": "processing", "total": total, "processed": 0, "current_file": "", "errors": [], "results": []}
         - `update_progress(task_id: str, processed: int, current_file: str)` -- updates processed count and current file name
         - `add_error(task_id: str, filename: str, error: str)` -- appends to errors list
         - `add_result(task_id: str, filename: str, status: str, page_count: int | None)` -- appends to results list
         - `complete_progress(task_id: str)` -- sets status to "completed"
         - `fail_progress(task_id: str, reason: str)` -- sets status to "failed"
         - `get_progress(task_id: str) -> dict | None` -- returns current progress or None
       - This is sufficient for single-user v1. No Redis needed.

    2. Create `app/services/document_service.py`:
       - Import get_parser_for_file from app.services.parsing
       - Import progress functions from app.services.progress
       - Import Document, DocumentStatus from app.models
       - Import async_session from app.database (for creating new sessions in background task)

       - `async def process_documents_batch(task_id: str, project_id: int, file_records: list[dict])`:
         - file_records format: [{"doc_id": int, "filename": str, "file_path": str, "file_type": str}]
         - init_progress(task_id, len(file_records))
         - Process documents SEQUENTIALLY (one at a time -- SQLite write contention, see research pitfall 4)
         - For each file_record:
           - update_progress(task_id, i, file_record["filename"])
           - Get parser via get_parser_for_file(file_record["filename"])
           - Call parser.parse(file_record["file_path"])
           - Create a NEW database session (async with async_session() as db) for each document update
           - Update the Document record in the database:
             - status = COMPLETED (or FAILED on error)
             - extracted_text = parsed.full_text
             - tables_json = json.dumps(parsed.tables)
             - metadata_json = json.dumps(parsed.metadata)
             - page_count = parsed.page_count
             - processing_time_ms = parsed.processing_time_ms
             - If error: status = FAILED, error_message = str(error)
           - Update Project counts (processed_documents += 1, or failed_documents += 1)
           - await db.commit()
           - add_result or add_error to progress
         - After all documents: complete_progress(task_id)
         - Update project status to READY (or FAILED if all docs failed)
         - Wrap entire function in try/except to catch unexpected errors and fail_progress

       **Critical:** Each document gets its own database session. Do NOT pass a session from the request handler to the background task (session lifecycle mismatch).
       **Critical:** Process documents sequentially, NOT concurrently, to avoid SQLite "database is locked" errors.

    3. Create `app/api/documents.py` with APIRouter(prefix="/projects/{project_id}", tags=["documents"]):

       - `POST /projects/{project_id}/upload` (see research FastAPI Multi-File Upload):
         - Accepts: files: list[UploadFile] = File(...)
         - Depends: db: AsyncSession = Depends(get_db)
         - Verify project exists (404 if not)
         - Define allowed_extensions = {".pdf", ".docx", ".xlsx"}
         - Create upload directory: Path(settings.upload_dir) / str(project_id), mkdir(parents=True)
         - For each file:
           - Check extension against allowed_extensions, skip unsupported
           - Generate safe filename: f"{uuid.uuid4().hex}_{file.filename}"
           - Save to disk using shutil.copyfileobj(file.file, open(dest, "wb")) -- stream to disk, NOT await file.read()
           - Create Document record in DB: project_id, filename (original), file_path (as_posix()), file_type, file_size, status=PENDING
         - Update project.total_documents and project.status = INGESTING
         - await db.commit()
         - Generate task_id = str(uuid.uuid4())
         - Build file_records list from saved documents
         - Start background task: asyncio.create_task(process_documents_batch(task_id, project_id, file_records))
         - Return UploadResponse: {"task_id": task_id, "uploaded": count, "skipped": skipped_count, "filenames": [names]}

       - `GET /projects/{project_id}/documents`:
         - Returns list of DocumentResponse for the project
         - Query: select(Document).where(Document.project_id == project_id).order_by(Document.created_at)

       - `GET /progress/{task_id}` (NOTE: this is at top level, not under project):
         - SSE endpoint using sse-starlette EventSourceResponse
         - Async generator that polls progress_store every 0.5 seconds
         - Yields JSON-serialized progress data as SSE events
         - Breaks when status is "completed" or "failed" or "unknown"
         - See research SSE Progress Streaming pattern

    4. Update `app/main.py`:
       - Add Jinja2Templates setup: `templates = Jinja2Templates(directory="app/templates")`
       - Include documents_router under /api prefix
       - Include progress endpoint (note: progress route might need its own router at /api level)
       - Ensure templates directory creation in lifespan
       - Add CORS middleware for localhost development (allow_origins=["*"] for v1 local-only)

    **Important notes:**
    - Use `shutil.copyfileobj(file.file, dest_file)` to save uploads (NOT await file.read() which loads into memory -- pitfall 3 and 6)
    - Store file paths as POSIX format in DB: str(path.as_posix())
    - Background task uses asyncio.create_task(), NOT FastAPI BackgroundTasks (we need the task to run independently)
